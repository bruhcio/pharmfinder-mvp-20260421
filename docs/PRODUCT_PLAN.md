# PharmFinder 서비스 기획서

## 1. 서비스 한 줄 정의

소비자가 약국 방문 전에 원하는 약의 보유 여부를 확인하고, 약국은 자동 연동 또는 간편 입력으로 재고를 업데이트하는 약국 재고 조회 플랫폼.

## 2. 해결하려는 문제

- 소비자는 특정 약품이 있는지 약국마다 직접 방문하거나 전화해야 한다.
- 약국은 재고 문의 전화가 많고, 응대 대비 전환율이 낮다.
- 재고 정보가 각 약국 내부 시스템에 갇혀 있어 소비자에게 노출되지 않는다.

## 3. 핵심 목표

- 소비자 관점:
  - 약 이름 검색으로 가까운 보유 약국 찾기
  - 사진 촬영 또는 업로드로 약품 찾기
  - 약국별 보유 품목 확인
  - 지도 기반으로 가까운 순 탐색
- 약국 관점:
  - POS/ERP가 있으면 자동 연동
  - 없으면 수기 입력 가능
  - 약품 자동완성으로 빠른 재고 입력
- 운영 관점:
  - 약품 마스터 데이터 표준화
  - 약국 마스터 데이터 정규화
  - 검색 정확도와 재고 최신성 관리

## 4. 사용자 유형

- 일반 소비자
- 보호자
- 만성질환자
- 외국 약명 또는 성분명만 알고 있는 사용자
- 야간/주말에 빠르게 약이 필요한 사용자
- 재고 문의가 많은 약국/약사

## 5. 핵심 기능 정의

### 5-1. 소비자 기능

- 약 이름 검색
  - 제품명, 브랜드명, 성분명, 별칭으로 검색
  - 자동완성 제공
- 사진 검색
  - 약 봉투, 약 상자, 낱알 사진 업로드
  - OCR 또는 비전 모델로 텍스트/형상 인식
- 지도 검색
  - 현재 위치 또는 지역 검색
  - 가까운 약국 순 정렬
- 약국 상세 조회
  - 보유 의약품 리스트
  - 재고 상태
  - 마지막 갱신 시간
  - 운영시간, 전화번호, 주소
- 사용자 액션
  - 전화 문의
  - 길찾기
  - 즐겨찾기
  - 최근 검색 기록

### 5-2. 약국 기능

- 약국 가입 및 인증
- 재고 대시보드
- 약품 자동완성 입력
- 바코드 입력
- 품절/소량/충분 상태 빠른 변경
- 운영시간 수정
- POS/ERP 연동 관리
- 최근 인기 검색 약 조회

### 5-3. 관리자 기능

- 약품 마스터 관리
- 약국 승인 및 상태 관리
- 비정상 재고 패턴 감지
- API 장애 모니터링
- 검색 로그와 미탐색 검색어 분석

## 6. 추천 추가 기능

- 대체 의약품 제안
  - 동일 성분 또는 유사 효능군 안내
  - 단, 의료적 판단으로 오인되지 않게 “약사 상담 권장” 문구 필요
- 재고 신뢰도 점수
  - POS 연동 여부
  - 마지막 갱신 시각
  - 최근 오차 신고율 반영
- 즐겨찾기 약국
- 재입고 알림
- 다국어 약품 검색
  - 영어 성분명, 해외 브랜드명 매핑
- 야간 운영 필터
- 소아용/시럽형/연질캡슐 등 제형 필터
- 조제 가능 여부 문의 기능
  - 단, 처방 조제 예약/판매 중개로 보이지 않게 설계 주의

## 7. 법적 검토

아래는 2026년 4월 21일 기준 확인한 법령/공식 자료를 바탕으로 한 제품 관점 정리이며, 실제 출시 전에는 의료·약사법 전문 변호사 검토가 필요합니다.

### 7-0. 기본 제품 포지션

- 본 서비스의 1차 포지션은 `의약품 재고 정보 전달 서비스`다.
- 핵심 목적은 소비자의 헛걸음과 반복 전화를 줄이는 것이다.
- 따라서 제품 구조도 판매, 주문, 예약 성격이 아니라 `조회`, `전화 문의`, `길찾기`, `방문 전 확인`에 맞춰 유지한다.

### 7-1. 상대적으로 안전한 영역

- 약국 위치, 운영시간, 전화번호, 보유 약품, 재고 상태 조회
- 약국 소개 및 길찾기 제공
- 소비자가 직접 약국에 전화하도록 연결
- 약국 내부 시스템과 재고 연동

### 7-2. 주의가 필요한 영역

- 온라인 결제, 예약 후 픽업, 주문 접수
  - `약사법 제44조` 및 약국 외 판매 관련 해석과 충돌 가능성이 큼
  - 서비스가 판매 중개 또는 원격 판매로 해석되지 않도록 주의 필요
- 전문의약품, 조제약 검색/광고 표현
  - 의약품 광고나 효능 표현 방식에 주의 필요
- 대체약 추천 문구
  - 치료적 효능을 단정하거나 의료적 판단처럼 보이면 위험
- 사용자 위치정보 수집
  - 위치 권한 동의, 개인정보처리방침, 보관 최소화 필요

### 7-3. MVP에서 피해야 할 것

- 앱 안에서 바로 의약품 주문/결제
- 비약사 주체가 특정 약 구매를 적극 알선하는 UX
- 전문의약품 홍보성 문구
- 사용자 증상 기반 자가진단 추천

### 7-4. 제품 문구 가이드

- “재고 조회 정보는 실제 방문 시점과 다를 수 있습니다.”
- “구매 가능 여부와 복약 상담은 반드시 해당 약국 약사에게 확인하세요.”
- “본 서비스는 의약품 판매자가 아니며, 판매를 중개하지 않습니다.”

## 8. 외부 데이터/API 권장안

### 8-1. 약품 데이터

- 식품의약품안전처 `의약품개요정보(e약은요)` OpenAPI
  - 제품명, 업체명, 효능, 사용법, 부작용 등 확보 가능
  - 공공데이터포털에서 신청 가능
- 추가로 품목기준코드 기반 마스터 테이블 구축 권장

### 8-2. 약국 데이터

- 국립중앙의료원 `전국 약국 정보 조회 서비스`
- 지역별 인허가 약국 공공데이터 병행
- 자체 약국 온보딩 시 정확 좌표와 실제 운영시간 재검증 필요

### 8-3. 지도

- 네이버 지도 JavaScript API v3
  - 한국 사용자 UX와 길찾기 전환 측면에서 적합

### 8-4. 사진 검색

- 1차: OCR
  - 라벨, 제품명, 성분명 텍스트 추출
- 2차: 비전 모델
  - 패키지 디자인, 낱알 식별 보완
- 출시 초기에는 “사진 검색은 보조 수단”으로 두고 텍스트 검색을 메인으로 유지 권장

## 9. 약국 재고 연동 전략

### 9-1. 1순위: POS/ERP/API 연동

- 약국에서 사용하는 재고/오더/청구 프로그램 사업자와 제휴
- 방식:
  - 주기적 pull API
  - webhook push
  - CSV/SFTP 배치
- 장점:
  - 정확도 높음
  - 약사 입력 부담 감소
- 리스크:
  - 사업자별 연동 난이도 상이
  - 제휴 없으면 접근 어려움

### 9-2. 2순위: 약사 수기 입력

- 빠른 재고 상태 토글
- 약품 자동완성
- 바코드 스캔
- 자주 찾는 약품 즐겨찾기
- 마지막 입력 시간 시각화

### 9-3. 3순위: 하이브리드

- 연동되는 품목은 자동 반영
- 누락 품목만 약사가 수정
- 충돌 시 약사 입력 우선 또는 시스템 우선 정책 명시
